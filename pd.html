Presence-Detector

Have you ever needed someone's help,
but you weren't sure if they are on a meeting, in a phone call or just listening to some music?
Well, i and many other of my Colleagues at work ask ourselves this exact question.
So i took it upon myself to stop this madness. Rather my Supervisor told to.
With his help, i built a little LED-Light, which sits on top of you monitor
and shines Green if you are available and red if you are in a meeting or on the phone.

How it works:

Hardware:

-3D-Printed Case
-Half of a Ping-Pong Ball
-Shelly RGBW2 LED-Controller
-Adafruit Button
-GekPower 12V Flat Round 5050 RGB LED
-Speaker Wire as Wire
-Welleman 5V to 12V Convertor
-USB A Cable for Power Delivery

Software:

So, this runs on 3 Seperate Scripts. The first one fills all the information about the shellys from our DNS-Server to a Database.
The second Script then requests the Status of any person from Cisco Webex and fills that Statusinformation into the Database.
Soon i want to add an integration into our Time-Management-System and then the script will only get the status if the person is clocked in,
This should save a lot of API resources. Then Finaly we have the thrid and final Script.
This one is a shared funtion library with an enum of all the different Presence-Statuses.
Because the Computer is faster to compare two numbers than an entire string.
In said Script it also has the Function to set the LED.
So when the second Script which gets the Status sees that the status has changed since last time, it will set the LED again.
